# WiFi Credentials и State Processing Implementation

## Обзор

Реализована система управления состоянием записывающего устройства с автоматической обработкой WiFi подключения на основе файла конфигурации `.creds`.

## Основные компоненты

### 1. **SDStorage расширение** (`src/sd_storage.hpp/cpp`)
Добавлены методы для работы с файлами:
- `read_file()` - чтение содержимого файла
- `write_file()` - запись в файл
- `file_exists()` - проверка существования файла

### 2. **Recorder::state** (`src/recorder.hpp/cpp`)
Изменено на статическую переменную для глобального доступа:
```cpp
static State Recorder::state = Recorder::WAITING_FOR_CREDS;
```

### 3. **StateProcessor** (`src/state_processor.hpp/cpp`)
Полностью реализована логика `process_waiting_for_creds()`:

#### Логика работы:

**Первый запуск:**
1. Проверяет наличие файла `/sdcard/.creds`
2. Если файл отсутствует:
   - Создаёт пустой файл с комментарием о формате
   - Запускает WiFi в режиме AP (точка доступа) с названием `SETUP_ME` без пароля
3. Если файл существует и НЕ пустой:
   - Парсит строки в формате `SSID:Password`
   - Пропускает пустые строки и комментарии (начинающиеся с `#`)
   - Добавляет сети в список для подключения
4. Если файл существует и пуст:
   - Запускает WiFi AP режим `SETUP_ME` без пароля

**Цикл подключения:**
- Последовательно пытается подключиться к каждой сети из файла
- Проверяет статус подключения
- При успешном подключении переходит в состояние `READY`
- Если все попытки исчерпаны и подключение не удалось:
  - Пользователь может переподключиться к AP и обновить файл `.creds`
  - При следующем цикле устройство снова попробует подключиться

### 4. **main.cpp** 
Добавлена инициализация и главный цикл обработки состояния:

```cpp
// После инициализации SD карты:
StateProcessor::Config sp_cfg{};
sp_cfg.process_interval_ms = 100;  // Проверка каждые 100ms
StateProcessor state_processor(sp_cfg);

// Основной цикл
while (true) {
    state_processor.process();  // Обрабатывает текущее состояние
    vTaskDelay(10 / portTICK_PERIOD_MS);
}
```

## Формат файла .creds

```
# WiFi credentials format: SSID:Password
# Lines starting with # are comments

MyNetwork:MyPassword
AnotherNetwork:AnotherPassword
HomeWiFi:SecurePassword123
```

## Состояния (Recorder::State)

1. **WAITING_FOR_CREDS** - Ожидание подключения к WiFi
   - Проверяет .creds файл
   - Пытается подключиться или запускает AP режим
   
2. **READY** - Устройство подключено и готово к работе
   
3. **RECORDING** - Идёт запись аудио
   
4. **SENDING** - Отправка данных на сервер

## Логирование

Все операции логируются с префиксом `StateProcessor`:
- Загрузка и парсинг `.creds`
- Попытки подключения
- Успешные подключения
- Переходы между состояниями

## Обработка ошибок

- SD карта не смонтирована → возврат `ESP_ERR_INVALID_STATE`
- Файл не найден → создание нового с пустым содержимым
- Ошибки парсинга → пропуск некорректных строк
- Неудачное подключение → попытка следующей сети из списка
